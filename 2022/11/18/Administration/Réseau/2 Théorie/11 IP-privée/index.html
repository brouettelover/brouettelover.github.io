Adresse Ip privée caractéristiques et Masque de sous réseaux

Brouettelover 2022-11-18 13:24:54
Categories： Réseau > Administration > Conception de réseaux Tags： IP IPv4 IPv6 Public

Les plages d’adresses privées IPV4 (32bits) sont définies sur 3 plages :
10.0.0.0 à 10.255.255.255 soit 10.0.0.0/8
172.16.0.0 à 172.31.255.255 soit 172.16.0.0/12
192.168.0.0 à 192.168.255.255 soit 192.168.0.0/16

Elles servent uniquement dans le réseaux internes c’est à dire des adresses situées localement (qui ne sont pas visible/accessible via internet)

Il existe dans les adresses privées plusieurs emploies pour l’envoie de requête:

-> Monodiffustion (unicast) adresse des plages d’adresses si dessus

-> Multidiffusion (multicast) de 224.0.0.0 à 239.255.255.255

-> Adresse de diffusion (broadcast) 255.255.255.255

L’adresse de diffusion: permet l’envoie de paquets à l’ensemble du réseau local.
L’adresse de multidifusion: permet l’envoie de paquets à un ensemble d’hôtes. Les groupes de multidiffusions sont définis soit manuellement ou directement par un protocole de couche 3 comme OSPF.

IPV6 (128 bits) bien plus longues que les adresses IPV4 s’écrivent sous la forment de 8 groupes de 2 octets écrit sous la forme Hexadécimal.

Exemple:
2A01:EF35:2421:4BE0:CDBC:C04E:A7AB:ECF3

Ils existent certaines règles pour diminuer la taille de notation des IPV6:

les bits à 0 précédent un chiffre d’un même groupe sont ignorés. Si un groupe d’octets est à 0 l’écriture est réduite à :: pour l’ensemble des premiers groupes qui se suivent sinon ils sont réduit à :0:

Exemple:
0A01:0000:0000:00C2:AF12:0123:0000:ECF4
s’écrit
A01::C2:AF12:123:0:ECF4

Ils existent 3 types d’adresses en IPV6:

-> Monodiffusion (identifie une interface)

-> Multidiffusion (permet l’envoie de paquets vers plusieurs destinations)
-> FF02::1 équivalent de 255.255.255.255 (broadcast)
-> FF02::2 contact l’ensemble des routeurs (ayan l’option activé IPV6 unicast-routing)

-> Anycast (Adresse attribuable à plusieurs périphériques, un paquet envoyé en anycast est acheminé vers le périphérique le plus proche)

Types d’adresses de monodiffusion IPV6 :
-> Monodiffusion global: similaire à une adresse IPV4 publique. elles sont uniques et routables sur internet. Ces adresses sont statiques ou dynamiques. soit 001 ou 2000::/3 (attribuées par l’ICANN)
Elles se décomposent en 3 parties :
-> Préfixe de routage global: Partie attribuée par le FAI sur /48 pour le moment.
-> ID de sous-réseau: utilisé pour identifier les sous-réseaux au sein du site d’une entreprise.
-> ID d’interface IPV6: Similaire à la partie Hôte d’une IPV4.

-> Link-local: Utiliser pour communiquer avec d’autres périphériques sur la même liaison locale. Elles ne sont pas routable au delà de la liaison. Elles servent uniquement sur un même sous réseau. FE80::/10

-> Adresse de bouclage: utilisé pour envoyer un paquet à soi-même. ::1/128
