DHCP serveur

Brouettelover 2022-07-06 18:55:37
Categories： Système > Administration Tags： DHCP Serveur

Installation sur un serveur Oracle Linux

Tout d’abord, il faut installer un serveur DHCP.
Pour se faire, j’utilise DNSmasq. Le choix étant fait pour certaines options que le service permet.

Attention, si un serveur dhcp est déjà présent sur le réseau. Il va rentrer en collision avec celui-ci. Il est donc nécessaire de désactiver celui déjà présent. Je vous conseil d’aller vérifier dans votre box internet sur son adresse de configuration afin de désactiver le service dhcp.

sudo yum install dnsmasq #install le programme si vous utilisez un serveur redhat,cent0s,oracle,...
sudo systemctl start dnsmasq #démarre le service
sudo systemctl enable dnsmasq #active le service au démarrage
interface=eno1,lo #interface fournissant le DHCP
port=0 #Port par défaut d'écoute
# DHCP range-leases
dhcp-range= eno1,192.168.0.150,192.168.0.200,255.255.255.0,6h #Range d'ip de 150 à 200 avec 6h de bail
# Gateway
dhcp-option=3,192.168.0.1 #Fournit l'adresse de la passerelle par défaut à tous les utilisateurs
# DNS
dhcp-option=6,192.168.0.100 #Fournit l'adresse du serveur DNS. J'utilise un DNS personnel mais vous pouvez utiliser des DNS public comme ceux de google sur les adresses : 8.8.8.8 - 8.8.4.4

Firewall

Dans le cas où vous utilisez Firewalld, il faut ajouter le service DHCP à la liste des services autorisés.

sudo firewall-cmd --zone=internal --add-service=dhcp --permanent
sudo firewall-cmd --reload

internal défini la zone local (lan) sur laquelle le pare feu autorise le service.
